# UI Design

Use this skill when work involves product UI, app screens, dashboards, websites, component design, UI review, or turning design guidance into implementation decisions.

## Workflow

1. Start with `core-loop.md` to define the user goal, primary action, layout system, hierarchy, states, and design-system follow-through.
2. For page structure, use `screen-framing.md`, `grid-discipline.md`, `spacing-hierarchy.md`, `alignment.md`, and `box-model-thinking.md`.
3. For visual decisions, use `visual-hierarchy.md`, `typography-scale.md`, `color-accessibility.md`, and `effects-discipline.md`.
4. For common product patterns, use the relevant pattern files:
   - `components-and-states.md`
   - `forms-and-inputs.md`
   - `pricing-ux.md`
   - `navigation-search.md`
   - `modals.md`
   - `hero-sections.md`
   - `cards.md`
5. Before finishing, use `design-system.md` and `critique-checklist.md` to check consistency, accessibility, responsiveness, states, and reusability.

## Operating Principles

- Prefer reusable systems over one-off visual choices.
- Tie visual hierarchy to the user's primary goal.
- Use grids, spacing tokens, type scales, and color semantics that developers can implement.
- Cover responsive behavior and interaction states before calling UI work complete.
- Keep accessibility checks explicit, especially contrast, focus, labels, and error states.
