# Core UI Design Loop

## Purpose

Use this loop before designing or implementing any screen. It keeps visual decisions tied to user goals and implementation constraints.

## Repeatable Practice

1. Define the user's primary goal.
2. Identify the primary action the screen should drive.
3. Choose the device/frame target first.
4. Lay out content using a grid and spacing scale.
5. Establish hierarchy with size, position, spacing, typography, and contrast.
6. Design in grayscale first when possible.
7. Add color, imagery, shadows, gradients, and interaction states.
8. Check accessibility, responsiveness, and edge states.
9. Convert repeated UI into reusable components.
10. Document decisions in a style guide or design system.

## Development Checklist

- The main action is obvious.
- The layout can be implemented with predictable spacing tokens.
- Empty, loading, error, disabled, and long-content states are covered.
- Mobile and desktop layouts are both considered.
- Repeated UI is componentized instead of duplicated.
