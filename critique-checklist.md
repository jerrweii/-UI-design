# UI Critique Checklist

## Purpose

Use this checklist to review a design before implementation or release.

## Repeatable Practice

Ask these questions:

- Is the primary user goal clear?
- Is the primary action obvious?
- Is the layout aligned to a grid?
- Is spacing consistent and meaningful?
- Is the typography scale consistent?
- Is the content hierarchy easy to scan?
- Is color used functionally, not only decoratively?
- Does contrast meet accessibility expectations?
- Are interactive states designed?
- Are empty, loading, error, disabled, and long-content states covered?
- Does the layout work on mobile and desktop?
- Are repeated patterns componentized?
- Are risky actions confirmed clearly?
- Is the UI simpler than it was before?

## Development Checklist

- Check the screen in grayscale.
- Check keyboard focus and tab order.
- Check touch target sizes.
- Check real data, long data, and missing data.
- Compare against existing components and style guide rules.
