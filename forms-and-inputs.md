# Forms and Inputs

## Purpose

Make data entry clear, forgiving, and easy to complete.

## Repeatable Practice

- Keep labels visible.
- Use placeholders as examples, not label replacements.
- Match input width to expected content length.
- Validate input clearly and show errors near the field.
- Use helper text for format requirements.
- Use checkboxes for multiple choices.
- Use radio buttons for one visible choice.
- Use dropdowns or selects when many options must fit in limited space.
- Make completion as easy as possible.

## Development Checklist

- Required and optional fields are clear.
- Error messages are specific and actionable.
- Inputs support keyboard and screen-reader use.
- Long forms are grouped into logical sections.
- Confirmation is required for risky or irreversible submissions.
