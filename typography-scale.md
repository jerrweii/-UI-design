# Typography Scale

## Purpose

Use type consistently so screens feel coherent and are easier to implement.

## Repeatable Practice

- Define font sizes, weights, line heights, and use cases.
- Use a limited set of text styles.
- Make headings prominent but proportionate to the surface.
- Use readable line height for body copy.
- Keep labels, helper text, and metadata distinct but legible.
- Reuse the scale instead of inventing sizes per component.

## Development Checklist

- Type styles are represented as tokens or reusable classes.
- Text does not overflow buttons, cards, tabs, or form fields.
- Long words and localized strings have enough room.
- Heading levels follow content structure, not just visual preference.
