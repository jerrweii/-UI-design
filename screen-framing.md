# Screen Framing

## Purpose

Choose the right canvas and information density before designing details.

## Repeatable Practice

- Start with the actual target device or viewport.
- For mobile, keep the flow focused and prioritize the most important task.
- For desktop, use the extra width for comparison, context, scanning, and secondary controls.
- Prefer scaling focused mobile ideas up over squeezing a dense desktop layout down.
- Decide whether the screen is a focused task, a browsing surface, a dashboard, a form, or a conversion page.

## Development Checklist

- Responsive breakpoints are known before implementation.
- The layout has a clear mobile strategy.
- Content priority changes are intentional across viewports.
- The first viewport communicates the main purpose and action.
