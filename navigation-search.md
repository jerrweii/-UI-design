# Navigation and Search

## Purpose

Help users know where they are, where they can go, and how to find content.

## Repeatable Practice

- Keep navigation labels simple and recognizable.
- Show the user's current location.
- Separate the main CTA from normal navigation items.
- Use dropdowns only when they reduce complexity rather than hiding important paths.
- Make search visible when discovery matters.
- Support recent searches, autocomplete, suggestions, clear/reset behavior, and no-results states.

## Development Checklist

- Active navigation state is visible.
- Navigation works at small desktop and mobile widths.
- Search has an empty state and no-results state.
- Search results update predictably and can be cleared.
- Keyboard users can navigate menus and search interactions.
